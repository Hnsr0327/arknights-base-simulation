# 明日方舟基建运转大模拟 (arknights_base_simulation)

输入**干员练度表**、**每日上线时刻(1~4次)**、**素材理智价值(config.json)**,
搜索产生**最大日均理智收益**的循环生产系统(布局 + 排班)。

```
干员练度表.xlsx ─┐
每日上线时刻   ─┼─►  优化器  ─►  最优布局(制造/贸易/发电配比) + 各设施排班 + 日均理智收益
config.json    ─┘
```

## 快速开始

```bash
source ../../../.venv/bin/activate          # 项目根的虚拟环境
cd active/playground/arknights_base_simulation

# 默认: 用 ~/Downloads/干员练度表.xlsx, 每日上线 08:00 / 22:00
python -m arknights_base_simulation

# 指定练度表 + 每日 4 次上线
python -m arknights_base_simulation /path/干员练度表.xlsx --logins 7,12,17,22

# 用官方『物品价值表.xlsx』实时载入素材理智价值(按物品id)
python -m arknights_base_simulation --logins 8,22 --values-xlsx /path/物品价值表.xlsx

# 自定义素材价值 / 产能常量
python -m arknights_base_simulation --config my_config.json --logins 8,22

# 机器可读输出
python -m arknights_base_simulation --logins 8,22 --json

# 逐日瞬态模拟: 按真实时间轴推进心情, 输出每日产出曲线(默认错峰=已运行基建)
python -m arknights_base_simulation --logins 8,22 --days 10
# 看新部署的心情爬坡(全员从指定初始心情同步出发)
python -m arknights_base_simulation --logins 8,22 --days 10 --initial-mood 12

# 手写多班次排班: 跳过优化器, 按给定 1~4 组排班轮换并做瞬态模拟
python -m arknights_base_simulation --logins 0,6,12,18 --shifts shifts_example.json --days 7

# 自动生成 2~4 组轮换排班: 中枢/制造/贸易按班次分配, 发电/会客/办公沿用同一批
python -m arknights_base_simulation --logins 0,6,12,18 --n-shifts 3 --days 7
```

## 三个输入

1. **练度表 xlsx** — 列固定: `干员名称, 是否已招募, 星级, 等级, 精英化等级, 潜能等级,
   通用技能等级, 1/2/3技能专精等级, χ/γ/Δ/α分支模组`。只取已招募干员;
   **精英化等级**决定解锁哪些基建技能。
2. **`--logins`** — 每日上线时刻(24h制, 1~4 个)。两次上线之间干员不可换班,
   决定了产能溢出和心情耗尽造成的损耗(见下)。
3. **`config.json`** — **素材理智价值**(`material_values_ap`, 收益估值核心)
   + 各设施产能/心情/电力常量。所有数值可改。

## 计算模型

### 数据来源
- `data/skills.json` (899 条) + `data/values.json` (oa 数值表) + `data/groups.json` (互斥组)
  全部来自一图流的 `buildingSkillFilter` / `logistics` 前端数据。
- 技能解锁: 记录 `phase<=精英化` 且 `level<=干员等级` 时可用。同一设施内,
  按一图流 `gr` 互斥组取已解锁最高阶版本(升级链如 ·α/·β 取最高), 不同组共存
  (如缇缇加工站两个独立技能)。
- **重要**: 一图流的 `logistics.js` 只是「养成成本计算器」, **不含产出/心情/产能公式**。
  本项目的产能、心情、电力、赤金→龙门币换算等**游戏机制已按 PRTS wiki (prts.wiki) 校准**,
  常量写在 config.json 里可改。关键数值(均为基础值/满级):
  - 工作位: 控制中枢5 / 制造3 / 贸易3 / 发电1 / 宿舍5(×4间) / 会客2 / 办公1 / 训练1+1 / 加工1
  - 制造站: 库存按*体积*54(中级作战记录占5→10.8件、赤金占2→27件); 基础耗时 中级作战记录180分、赤金72分;
    实际产速 = 基础 ×(1+总生产力%/100), 每名非涣散驻员提供基础生产力+1%
  - 贸易站: 1赤金=500龙门币; 3级普通赤金订单按 PRTS 概率 2/3/4 赤金单=30%/50%/20%
    取长期期望(2.9赤金、1450龙门币、203.4分/单), 订单上限10; 每名非涣散驻员提供基础订单效率+1%
  - 心情: 上限24, 工作基础消耗1.0/h; 制造/贸易等同房间人数基础减免(1人0、2人−0.05、3人−0.1);
    控制中枢每名工作干员−0.05, 技能再加减
  - 宿舍恢复(PRTS公式): 基础/h = 1.5+0.1×等级 + 0.0004×min(氛围,1000×等级);
    L5空房2.0/h、满氛围5000达4.0/h(config 默认 level5+ambiance5000)。恢复干员技能分
    **全员/单体他人/自身**三类: 同类取最高、类别间叠加; **自身型只补恢复位自己, 不加速来休息的干员**
  - 电力: 发电站满级供电270/座; 制造/贸易/会客/办公/训练各耗60、加工10、宿舍65; **标准布局需≥3发电站**
  - 无人机: 全基建单一共享池, 基础≈10/h, 发电站非涣散驻员基础+5%充能, 后勤充能%再加速
  - 会客室/办公室: 会客室线索20小时/份、自有库上限10(按上线收取间隔封顶)、有进驻干员时每日4:00发放1份,
    并计宿舍总氛围、干员稀有度/精英阶段、会客室等级、非涣散+5%;
    办公室公招刷新12小时/次、人力资源余量上限3, 有非涣散驻员基础联络效率+5%

### 排班 / 心情 (按上线时刻离散结算)
- 上线时刻把一天切成若干**间隔** gap。心情上限 `cap`(24)。工作干员每小时净消耗
  `d = base_drain + 技能心情消耗`。
- 某干员在长度 g 的间隔里有效工作时长 `= min(g, cap/d)` —— 心情耗尽即停产。
  全天有效工作占比 `frac = Σ min(g, cap/d) / 24`。**上线越频繁、心情消耗越低 → frac→1**。
- 可持续性: 工作总消耗 vs 宿舍总恢复, 不足则按比例下调收益并告警(需要更多宿舍)。

### 各设施
| 设施 | 产出 | 公式要点 |
|---|---|---|
| 制造站 | 作战记录(经验)/赤金/源石碎片 | `产率 = base × (1 + 总生产力%/100)`, 受库容溢出限制 |
| 贸易站 | 龙门币/合成玉 | 默认 `strategy=gold`: 赤金订单消耗制造站的赤金, 3级普通订单按 PRTS 2/3/4 赤金单概率取期望; **没有赤金就出不了单**, 受订单上限溢出限制。可选 `strategy=orundum`: Lv3开采协力按 2源石碎片→20合成玉、2小时/单结算 |
| 发电站 | 无人机充能 | `base × (1 + 充能%/100)` |
| 控制中枢 | 全局加成 | 所有制造站生产力% / 贸易站订单效率% / 心情恢复, **同种效果取最高** |
| 宿舍 | 心情恢复 | `base + 入驻恢复技能(oa/100)` |
| 会客室 | 线索 | 20小时/份 + PRTS基础加成 + 后勤技能, 按 `线索价值` 估值 |
| 办公室 | 公招联络 | 12小时/次 + 驻员基础5% + 后勤技能, 按 `hire.ap_per_refresh` 估值 |
| 加工站/训练室 | 副产品/专精提速 | 加工站按 `craft_category` 匹配加工类别后估值期望副产品; 训练室默认0, 可用 `target_mastery_level`/`target_branch` 门控训练条件额外项 |

> **赤金↔龙门币链**: 这是 `制造站(产赤金) → 贸易站(卖赤金换龙门币)` 的耦合。
> 若你把 `material_values_ap.龙门币` 调高(认为龙门币更稀缺), 优化器会自动从经验流
> 转向龙门币流, 并多建贸易站; 此时上线频率影响更明显(订单上限易溢出)。
> **源石碎片↔合成玉链**: PRTS Lv3 贸易站“开采协力”可通过 `trading.strategy=orundum`
> 启用, 按 `制造站(产源石碎片) → 贸易站(2源石碎片换20合成玉)` 结算。源石碎片制造线默认仍关闭,
> 需要在 `manufacture.lines` 中显式配置, 避免默认优化误选通常净值为负的碎片链。

### 逐日瞬态模拟 (`--days N`, simulate.py)
稳态评估给的是『假设每个间隔开始时心情满血』的乐观日均(上界)。`--days N` 则按真实时间轴
**逐 gap 推进每名干员的心情**, 跟踪初始爬坡与可持续平衡:
- **占空循环**: 生产干员满心情上岗, 心情耗尽(<floor)下班进宿舍恢复, 回满再上岗(带迟滞)。
  休息期间该工位空缺(固定排班无替补 -> 产量真实下降)。长期工作占比由『净消耗 vs 宿舍恢复』决定。
- **库存不跨日**: 每次上线收菜清空, 故产量按 gap 结算; 唯一跨日累积的状态是心情。
- **初始心情**: 默认**错峰**(代表已运行基建, 各干员心情铺开 -> 平滑); `--initial-mood M` 则全员
  从 M 同步出发(展示新部署的爬坡与同步周期波动)。
- 输出每日产出曲线 + `可持续日产(后半程均值)`, 通常为稳态乐观值的 ~75–85%——差额正是稳态忽略的轮换损耗。
- 局限: 固定排班无替补, 工位休息空缺是该方案的保守下界; 真实玩家有富余干员轮换可填平。

### 手写多班次排班 (`--shifts shifts.json`)
`--shifts` 接受 1~4 组排班 JSON, 直接按每日上线间隔轮换, 不再调用优化器。每组可填写
`control`、`manufacture`、`trading`、`power`、`meeting`、`hire`、`dormitory`、`training`;
仓库内的 `shifts_example.json` 和 `shifts_gongsun.json` 可作格式参考。

多班次模式下, 当班干员上岗; 非当班干员自动进入宿舍恢复, 下次上岗沿用恢复后的心情。
这用于检查手写 243/252 轮换方案在真实心情时间轴下是否可持续。

`--n-shifts N` 会由优化器自动生成 2~4 组轮换排班。当前策略只轮换中枢/制造/贸易;
发电站、会客室、办公室沿用单班优化结果中的同一批干员。贸易特殊机制/暖机干员仅提高候选排序,
不会被强制复制到每个班次。

### 优化器
枚举 `制造:贸易:发电` 对 9 个生产位的所有瓜分 → 每种布局贪心排班(控制中枢全局加成
优先 → 制造/贸易边际收益 → 发电/会客室/办公室 → 逐房间选最优生产线 → 宿舍补心情)
→ 轻量局部搜索替换生产位 → 取日均理智最高的布局。整体 <1s。

## 数据来源 / 校准
- **素材理智价值**: 一图流『物品价值表.xlsx』(经验/龙门币/赤金/源石碎片/无人机/合成玉等)。
  `valuetable.py` 按物品id映射, `--values-xlsx` 可实时载入。
- **基建机制常量**: PRTS wiki (prts.wiki) —— 工作位/库存体积/配方耗时/赤金换算/心情/电力,
  见上节『关键数值』, 全部写在 config.json。
- **赤金是中间产物**: 只有被贸易站卖出才变现龙门币; 未卖出的赤金默认不计收益
  (`trading.unsold_gold_value_factor=0`), 迫使制造↔贸易闭环平衡。设为 1.0 则按素材价值全计。

## 动态联动 (synergy.py)
两遍计算: 先建上下文(干员位置 / 各派系基建·房间人数 / 各抱团资源池存量), 再据此解析每个技能:
- **条件配对/在场**: 「当与X在同一个设施」「X在基建内」—— 满足才计; *纯条件*技能未满足则不计
  (避免把如 德克萨斯·恩怨 的 +65% 名义值无条件高估)。含无条件基底的(如 蕾缪安 +20% 基底
  / 与能天使额外 +25%)只门控条件部分。
- **派系计数**: 「每名/每有1名 X派系干员 +y%」—— 按 `data/factions.json`(官方 character_table 派系)
  的真实人数(受上限)缩放。怪物猎人小队联动暂未进官方镜像, 在 synergy.py 手动补充成员。
- **抱团资源系统**(逐条对照描述硬编码): 人间烟火(岁)、思维链环+感知信息(迷迭香系)、无声共鸣、
  热情值(邦多利)、魔物料理(莱欧斯小队)、木天蓼(怪猎)、乌萨斯特饮/情报储备(彩虹小队)、巫术结晶。
  含转化链(宿舍人数→感知信息→思维链环/无声共鸣; 人间烟火→巫术结晶)。
- 优化器: 默认开满 **4 间宿舍**(供抱团资源/宿舍联动); 团队感知的 `_synergy_pass` 会把配对伙伴
  (如 拉普兰德↔德克萨斯)安置到同一房间以激活联动。

### 特殊贸易干员(改写赤金订单, config.json `trading.special_ops`)
PRTS 官方技能文本实装, 按 3 级站赤金订单概率取长期期望:
- **但书·违约单**: 把交付<4 的原生赤金单 +2(精2)/+1(精0)赤金交付, 报酬同步每赤金+500;
  出单时间不变, 继承普通/高品质原订单概率, 赤金消耗随违约单增加(供应不足则空转)。
- **龙舌兰·投资**: 原生≥4 赤金订单龙门币 **+500(精2)/+250(精0)**, 不额外耗赤金;
  普通3级站按原生4赤金单20%概率折成期望加成。
  **与但书互斥**——但书造的"违约订单不视作赤金订单", 同站则龙舌兰加成全失效。
- **U-Official·天真的谈判者**: 尤里卡订单固定赤金交付数为2、报酬1000龙门币, 但订单时长等其他属性继承原订单。
- **巫恋·低语**(精2): 同站**其他干员**订单效率归零, 这些其他干员**每人自身 +45%**(不含巫恋自身),
  三人站 = 其他 2 人×45% = **+90%**; 全员心情消耗 +0.25/h; 解放搭档席位给龙舌兰/柏喙等不靠效率的干员。
- **高品质贵金属订单**: 单个 α/β、两个 α 提质技能按 PRTS 3级站概率表折成期望订单
  (α: 2/3/4赤金单=15%/30%/55%, 3h达峰; β: 5%/10%/85%, 5h达峰),
  按上线窗口日均化 warm-up, 并联动龙舌兰的原生4赤金单概率、但书的原生<4赤金单概率。
- 优化器据此把这三人纳入贸易候选, 实战会摆出社区双体系("一站但书违约单 + 一站巫恋+龙舌兰")。

## 已知简化
- 心情阈值型(夕/令等)在逐日瞬态或显式 `mood_override` 下按当前心情分支; 稳态无心情态时默认高心情分支。
- 派系计数/资源/人数缩放会**保留同技能的无条件基底**(base + 缩放项)。配对/在场型额外加成
  (如 蕾缪安 与能天使额外 +25%、摩根 与推进之王额外 +35%)按 PRTS 文本门控, 且条件对象需处于工作状态。
- 会客室(线索→信用)、办公室(公招刷新)、加工站/训练室产出非理智素材, 按*保守*估值。
  加工站默认 `crafts_per_day=0`, 不计产值; 若配置 `crafts_per_day`、`craft_category`、
  `base_byproduct_chance`、`ap_per_byproduct`, 则只把匹配加工类别的副产品技能计入期望理智。
  若再配置 `craft_lmd_cost`, 瑕光等“减免龙门币消耗”技能会按实际加工次数折算节省龙门币。
  莱伊/洛洛/蚀清等“副产品必定为X组”会在 `fixed_byproduct` 明细中标出, 不改变通用副产品数量估值。
  `craft_category` 可取 `any/elite/skill/chip/building/alloy/polyester/oriron/rock/crystal/ketone/device`。
  它们是固定房间, 不影响经济区配比。
  训练室默认 `ap_value_per_hour=0`; 若估值训练, `target_mastery_level` 与 `target_branch` 会扣除不匹配的
  “专精至N级/训练位分支”条件额外速度和心情消耗, 且训练室等级按 PRTS 限制可执行的最高专精等级。
- 无人机按 PRTS「每架减少 3 分钟基础耗时」在制造站/贸易站间选择最高边际收益目标
  (见 `power.drone_minutes_per_drone`; 控制中枢Lv3解锁无人机协助), 不再被生产力%/订单效率%二次放大, 非按素材价值估值。
- 普通、单个α/β、两个α提质赤金订单按 PRTS 概率与达峰时间取长期期望, 未逐笔随机模拟。
- 心情「指定某干员」等细节做近似; PRTS 标注不确定项(高级作战记录耗时、精确线性公式)采用社区通行算法。
- 控制中枢「指定贸易站/制造站」类技能按目标房间定向结算, 不作为全局效率近似。

```

  roster.py     练度表解析
  skills.py     技能库 + 解锁/互斥组解析
```
